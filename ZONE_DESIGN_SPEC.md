# Fire-Reader Zone & Level Design Specification

## Design Philosophy

Based on research into educational game design for 4-5 year olds:

**Novelty drives engagement.** Research shows novelty produces a clear inverted-U relationship with intrinsic motivation — children play more when environments keep changing. Every level in Fire-Reader gets a unique zone. No reuse.

**Familiar → Adventurous → Epic.** The escalation follows developmental engagement patterns: start with settings a child recognizes from daily life (kitchen, bakery, school bus), graduate to exciting real-world settings (construction site, lighthouse, ferry), then reward mastery with fantasy/spectacular zones (pirate ship, volcano, rocket). This "earned spectacle" keeps the child pushing forward.

**Frequent wins, short sessions.** Each level is 8-12 minutes. With 38 levels, that's 38 boss fights, 38 victory celebrations, 114 possible stars. More achievement moments = more motivation to return.

**The "big boss" finale.** The Castle Dragon remains the ultimate final boss — the narrative goal visible from the start. Everything builds toward defeating the dragon. The sentence levels before it escalate the setting drama so the dragon fight feels earned.

Sources:
- [Serious Educational Games for Children: A Comprehensive Framework (2024)](https://pmc.ncbi.nlm.nih.gov/articles/PMC10963373/)
- [Designing for Challenge in Large-Scale Adaptive Literacy Games](https://bera-journals.onlinelibrary.wiley.com/doi/10.1111/bjet.13146)
- [Is Difficulty Overrated? Novelty and Intrinsic Motivation in Educational Games](https://www.researchgate.net/publication/313477156)

## Boss Sprite Prompt Template (Corrected)

### The Problem with the Original Prompt

The original prompt said "flame flicker, hood wiggle, blink, gentle wobble" — too similar for the image generator, which collapsed them into 2 distinct states (frames 1-2-3-6 identical, 4-5 identical).

### Why NOT to use dramatic poses

The game engine already handles all mechanical movement on the boss sprite:
- **Pulse tween**: scale oscillation 1.0↔1.03 (continuous breathing)
- **Recoil**: x-shift 8-23px on Easy/Hard hits
- **Dodge**: x-shift -10px on Fail
- **Tint flash**: white/gray overlay on hits
- **Defeat**: spin 360° + shrink to 0 + fade

The sprite animation plays at **2fps with yoyo** (0→1→2→3→4→3→2→1, ~4s cycle). If frames have large size/position changes, these stack with the engine's tweens and create visual chaos.

### The Correct Approach

**Sprite frames = organic flame flicker. Game code = mechanical reactions.**

The base object (car hood, oven, scarecrow body, etc.) stays **identical** across all frames. Only the **flame portion** changes shape — like a real fire flickering in the wind. Eyes get 1 blink frame for personality.

Frame 5 (the 6th cell) is reserved as a **hit-reaction frame** — not part of the standard animation loop, available for programmatic use (e.g., briefly show frame 5 when the boss takes a hit).

### Template

```
Bright 2D storybook cartoon for a preschool learning game. Clean thick
outlines, simple flat shading with gentle gradients, minimal texture,
high readability, no photorealism, no 3D render look. Friendly,
non-scary tone. Consistent line weight and consistent palette across
all assets. No text, no logos, no watermarks, no UI elements.

3x2 sprite sheet, transparent background, 1536x1024, six 512x512
cells. No overlap between cells.

Boss: "[ZONE] Fire" — [DESCRIPTION OF THE FIRE CHARACTER].
The base object (body, structure) is IDENTICAL in all six frames.
Only the flame shapes change between frames.

Six frames in reading order (left-to-right, top-to-bottom):
1. FLICKER A — flames curve gently to the LEFT as if blown by a breeze,
   eyes open, relaxed expression
2. FLICKER B — flames stand TALL and centered, pointing straight up,
   eyes open, same expression
3. FLICKER C — flames curve gently to the RIGHT (mirror of frame 1),
   eyes open, same expression
4. FLICKER D — flames SPREAD LOW and wide, shorter and broader than other frames,
   eyes open, slight grin
5. BLINK — same flame shape as frame 2, but eyes CLOSED (happy squint),
   gentle smile
6. HIT REACT — same flame shape as frame 4, but eyes WIDE and round (startled),
   open-mouth surprise, slightly brighter/whiter flame color

Character body is the same size and position in every cell. Only the
flame silhouette and eyes change. Each cell: character centered,
consistent scale relative to cell bounds.
```

### Why this works
- **Frames 0-3** have four distinct flame DIRECTIONS (left, up, right, spread) — clearly different silhouettes from each other, but the character stays the same size
- **Frame 4** is distinguishable by the closed eyes (blink), even though the flame shape is reused
- **Frame 5** is distinguishable by wide startled eyes + brighter color (hit reaction)
- The base object never moves, so the animation layers cleanly with the engine's position/scale/tint tweens
- The yoyo loop (0→1→2→3→2→1→0) produces a natural flame-sway cycle

| Level | Zone | Tier | Word Count | Example Sentences |
|-------|------|------|-----------|-------------------|
| 30 | Circus Tent | 1 | 3-4 words | "A big bug ran." / "We can get wet." |
| 31 | Helicopter | 2 | 4 words | "The duck went in." / "The barn is far." |
| 32 | Speedboat | 3 | 4-5 words | "Dad had a red cap." / "I can run fast." |
| 33 | Pirate Ship | 4 | 5 words | "The frog is not here." / "She said yes to me." |
| 34 | Monster Truck | 5 | 5-6 words | "The kid can jump fast." / "He can kick the ball." |
| 35 | Jungle | 6 | 5-6 words | "We will stop and rest." / "The bird is in the nest." |
| 36 | Volcano | 7 | 6-7 words | "The camp is near the pond." / "The dog can sniff the mud." |
| 37 | Rocket | 8 | 6-7 words | "The cat can nap on the bed." / "We can play in the yard." |
| 38 | Castle | Final | Mixed 4-7 | Best-of selection across all tiers |

Each sentence level uses HP=10, drawing from a pool of ~15-20 sentences at its tier.

The current 50 sentences cover tiers 1-7. Additional sentences should be written for tiers 7-8 and the final boss to fully populate all 9 levels (target: 15-20 sentences per tier, ~150 total).
